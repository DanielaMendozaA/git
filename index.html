Explicando algunos conceptos de GITHUB

¿Qué es un commit?

Es una captura instantánea de los cambios que se han realizado en tu proyecto, es decir, cuando haces un commit estás guardando los cambios que has hecho en tu repositorio local. El comando más comúnmente utilizado para hacer un commit es git commit -m "mensaje descriptivo"

Los commits son fundamentales en el uso de Git. Puedes pensar en un commit como una captura de tu proyecto, donde se crea una nueva versión de ese proyecto en el repositorio. Dos características importantes de los commits son: Puedes recordar los cambios a los que se les hizo commit en una fecha posterior, o revertir el proyecto a esa versión. Si varios commits editan diferentes partes del proyecto, no se sobrescribirán entre sí aunque los autores de los commits no se conozcan entre sí.

Las ramas se utilizan para desarrollar características, corregir errores, o experimentar con nuevas ideas en un área contenida de tu repositorio. Cuando creas un repositorio en GitHub, se crea una rama por defecto llamada “main” o "master". Puedes crear ramas adicionales a partir de cualquier rama existente y cambiar entre ellas para trabajar en diferentes partes de tu proyecto.

Aquí tienes un ejemplo de cómo crear una nueva rama llamada mi-rama y cambiar a ella: git branch mi-rama git checkout mi-rama El primer comando crea una nueva rama llamada mi-rama. El segundo comando cambia tu espacio de trabajo a la nueva rama. Una vez que has terminado de hacer cambios en una rama, puedes fusionarla con otra rama utilizando una solicitud de extracción. Esto te permite revisar y combinar los cambios de una rama con otra de manera ordenada

¿Qué es y para que sirve Push?

Es un comando que te permite subir los commits desde tu rama (branch) local en tu repositorio Git local al repositorio remoto. Para poder subir a tu repositorio remoto, debes asegurarte de hacer commit a todos tus cambios al repositorio local. La sintaxis de este comando es la siguiente:
git push (nombre del repositorio) (nombre de la rama)

Existen diferentes opciones que puedes pasar a este comando. Por ejemplo, si quieres subir todos tus cambios al repositorio remoto con todas sus ramas, puedes utilizar: git push --all (/NOMBRE-REMOTO) Donde --all es la bandera que señala que quieres subir todas las ramas al repositorio remoto y NOMBRE-REMOTO es el nombre del repositorio remoto al cual las quieres subir.

Si quieres ignorar los cambios locales hechos al repositorio Git en Github, puedes usar el comando --force para subir ignorando esos cambios: git push --force (NOMBRE-REMOTO) (RAMA-REMOTA)

¿Qué es y para que sirve ADD?

Este comando se utiliza para agregar cambios en el directorio de trabajo al área de preparación (también conocida como índice) para la próxima confirmación. En otras palabras, le dice a Git qué cambios quieres incluir en la próxima confirmación. La sintaxis básica del comando es la siguiente: git add (ruta-del-archivo) Este comando puede ser ejecutado varias veces antes de una confirmación. Sólo agrega el contenido de los archivos especificados al momento que el comando add es ejecutado; si quieres que cambios subsecuentes sean incluidos en la próxima confirmación, entonces debes correr git add nuevamente para agregar el nuevo contenido al índice. El comando git add no agregará archivos ignorados predeterminados. Si cualquier archivo ignorado fue explícitamente especificado en la línea de comandos, git add fallará listando los archivos ignorados. Es importante recordar que antes de poder usar git add, debes tener un repositorio Git local con cambios realizados. Si aún no has hecho cambios, git add no tendrá ningún efecto.
